## Context

O-YOGA瑜伽原现有一个面向学员的静态课程网站，包含 13 门课程、85 节课时、50+ 体式数据。所有内容由 `build.js` 从 `data/` 目录的 JS 数据文件生成纯静态 HTML。

现有数据中已包含丰富的教学信息（sequence 时间分配、keyPoints 教学要点、safetyNote 安全提示），但这些数据在学员页面中只是辅助展示，没有针对教练的使用场景做优化。

教练的核心使用场景是：**课前备课**（在家/办公室看电脑）和**课堂参考**（教室里用手机/平板快速翻阅口令）。

## Goals / Non-Goals

**Goals:**
- 复用现有 lesson/pose 数据，零内容成本生成教练手册
- 口令卡片可在手机上单手快速翻阅，适合教室场景
- 时间线视图帮教练掌控课堂节奏
- 体式速查突出教练关注的对齐/纠错/辅助信息
- 构建流程集成到现有 build.js，一次构建同时生成学员页和教练页

**Non-Goals:**
- 不做用户登录/权限系统——教练手册和学员页面一样是公开静态页
- 不做实时计时器/倒计时功能——只提供时间线可视化参考
- 不做课程编辑/自定义——内容由数据文件统一管理
- 不做打印版 PDF 导出——聚焦移动端屏幕阅读体验
- 不新增数据字段——完全基于现有 lesson 和 pose 数据生成

## Decisions

### 1. 文件结构：`/instructor/` 独立目录

教练手册页面放在 `/instructor/` 目录下，与现有 `/courses/` 和 `/lessons/` 平行。

- `instructor/index.html` — 教练手册首页
- `instructor/<course-id>-<num>.html` — 每节课的教练详情页

**理由**：与学员页面物理隔离，各自独立演进，不影响现有 URL 结构。比在现有 lesson 页面上加 tab 切换更简单、更不容易破坏现有功能。

**备选**：在 `/lessons/` 页面内加教练视角 tab——放弃，因为会增加现有页面复杂度，且两端受众使用模式完全不同。

### 2. 口令卡片：Swipe 翻页式交互

将 lesson 的 sequence 数组转化为全屏卡片组，教练左右滑动（或点击箭头）逐步翻阅。每张卡片只显示当前步骤的：
- 步骤名称和持续时间
- 教学引导文字（note 字段）
- 当前步骤在整体中的进度条

**理由**：教练在教室里需要「瞥一眼就知道下一步做什么」，全屏单步显示比列表滚动更适合这个场景。触屏滑动操作也更适合双手可能正在示范的情况。

**实现**：纯 CSS scroll-snap + touch event，无需引入 Swiper 等库，保持零依赖原则。

### 3. 体式速查：复用 poses.js + 增强教练注释

体式卡片基于现有 `poses.js` 的 SVG 坐标数据生成图示，但在卡片上增加教练关注的信息：
- 从 lesson 的 keyPoints 中提取与该体式相关的对齐要点
- safetyNote 中的禁忌信息

**理由**：避免新建体式数据源，保持单一数据来源。教练需要的额外信息（对齐、纠错）可以从现有 keyPoints 字段中智能提取。

### 4. 时间线：线性可视化 sequence

将 sequence 的各步骤按 duration 比例渲染为水平时间条。每段标注步骤名和时长，点击跳转到对应口令卡。

**理由**：教练需要一眼看到整节课的节奏分配。线性时间条是最直观的表达，也是最简单的实现。

### 5. 构建扩展：build.js 追加教练手册生成

在现有 build.js 的 `generateLessonPages()` 之后，追加 `generateInstructorPages()` 函数。复用相同的数据加载逻辑，使用独立的 HTML 模板。

**理由**：保持单一构建入口，一次 `node build.js` 生成所有页面。

### 6. 设计风格：沿用主站配色 + 深色模式偏好

教练手册沿用主站的 sage/terracotta/cream 配色系统和字体选择，但口令卡片提供深色背景选项——教室灯光昏暗时手机屏幕不刺眼。

**理由**：品牌一致性 + 实际使用场景考量。深色模式通过 CSS 变量切换实现，不增加复杂度。

## Risks / Trade-offs

- **口令卡内容密度**：sequence 的 note 字段长度不一，有的详细有的简略，卡片显示效果可能参差不齐 → 设定最大显示行数，超长内容可展开查看
- **体式关联提取**：从 keyPoints 自动提取体式相关信息可能不够精准 → 初版直接展示该课时的所有 keyPoints，不做智能过滤
- **85 个教练页面的构建时间** → 纯字符串模板拼接，预计 < 1 秒，无风险
- **深色模式切换状态持久化** → 使用 localStorage 记住偏好，无需后端
