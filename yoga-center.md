# Yoga Center — 瑜伽练习与教学平台

## 项目定位

面向瑜伽老师和学员的内容型平台，以知识库和教练展示为核心，逐步扩展为完整的教学辅助系统。

**技术栈**：Next.js + MDX  
**产品形态**：内容驱动的静态优先网站，支持多教练入驻  

---

## 用户角色

| 角色 | 核心诉求 |
|------|---------|
| **瑜伽老师** | 展示个人品牌、管理教案、使用知识库辅助备课 |
| **学员** | 浏览教练信息、学习瑜伽知识、查找适合自己的练习内容 |
| **访客** | 了解平台、浏览公开内容 |

---

## 功能规划

### 第一期：内容基座

#### 1. 教练个人主页

多教练入驻机制，每位老师拥有独立的展示页面。

**教练 Profile**
- 个人简介（教学理念、背景故事）
- 资质认证（RYT 200/500、E-RYT、专项认证等）
- 擅长流派标签（阿斯汤加、流瑜伽、艾扬格、阴瑜伽等）
- 教学经历 / 学习经历时间线
- 头像与形象照

**课程展示**
- 当前开设的课程列表（名称、难度、时长、简介）
- 课程分类（常规课 / 工作坊 / 私教 / 线上课）
- 上课地点与时间信息

**联系方式**
- 社交媒体链接（微信、小红书、Instagram 等）
- 预约 / 咨询入口

**教练目录页**
- 所有入驻教练的列表/卡片展示
- 按流派、地区、资质筛选

#### 2. 瑜伽知识库

多维度并行的结构化知识体系，各板块独立但可互相关联。

**体式库（Asana）**
- 体式详情页：梵文名 / 中英文名、分类标签
- 动作要领与对齐要点（alignment cues）
- 辅助变体（使用瑜伽砖、伸展带等）
- 进阶 / 简化变体
- 禁忌症与注意事项
- 涉及的肌肉群与关节（关联解剖板块）
- 所属序列推荐（关联课程编排）
- 配图 / 示意图

**体式分类维度**
- 按类型：站立、坐姿、前屈、后弯、扭转、倒立、平衡、手臂支撑
- 按难度：入门 / 中级 / 高级
- 按功效：开髋、开肩、核心激活、脊柱灵活性等
- 按流派来源

**呼吸法（Pranayama）**
- 呼吸法详情：名称、步骤、要点
- 功效说明与适用场景
- 练习时长与频率建议
- 注意事项与禁忌

**冥想（Meditation）**
- 冥想类型介绍（正念、瑜伽睡眠、脉轮冥想等）
- 引导步骤
- 适用场景与配合建议

**瑜伽哲学（Philosophy）**
- 瑜伽经典文本导读（《瑜伽经》《薄伽梵歌》等）
- 八支瑜伽体系
- 五大元素、三质（Gunas）等核心概念
- 脉轮系统（Chakra）

**解剖学基础（Anatomy）**
- 瑜伽相关的功能解剖知识
- 主要肌肉群与骨骼结构
- 常见运动损伤与预防
- 体式中的生物力学分析

**知识关联**
- 体式 ↔ 解剖（哪些肌肉参与）
- 体式 ↔ 呼吸法（配合使用）
- 体式 ↔ 流派（属于哪个传统）
- 体式 ↔ 序列（常见搭配组合）

---

### 第二期：教学工具

#### 3. 课程编排工具

帮助老师从体式库中组合编排课程序列。

- 从知识库拖选体式，组合为课程序列
- 序列模板（如：阿斯汤加初级序列、经典流瑜伽编排）
- 自定义序列的保存与复用
- 按目标编排（开髋专题、后弯进阶、修复课等）
- 课程时长估算
- 难度自动评估
- 序列可视化预览

#### 4. 教案管理

老师创建、存储、复用自己的教学方案。

- 教案 = 序列 + 教学备注 + 口令提示 + 音乐建议
- 教案模板库（可参考也可自创）
- 教案标签与分类管理
- 教案版本历史
- 私有 / 公开 / 仅同行可见的可见性控制

---

### 第三期：互动与社区

#### 5. 学员功能

- 浏览知识库与教练页（无需登录）
- 学员账户与个人资料
- 课程预约 / 报名
- 练习记录 / 打卡日志
- 收藏体式与课程序列
- 学习进度追踪

#### 6. 反馈与评价

- 学员对课程的评价
- 教练收到的学员反馈汇总
- 体式讨论 / 问答

## 内容数据模型（核心实体）

```
Teacher（教练）
├── profile: 个人信息、资质、简介
├── courses: 开设课程列表
├── lessonPlans: 教案集合
└── socialLinks: 社交媒体

Asana（体式）
├── names: 梵文/中文/英文名
├── category: 分类标签（多维度）
├── instructions: 动作要领
├── variations: 变体列表
├── contraindications: 禁忌症
├── anatomy: 关联肌肉/关节
└── media: 图片/视频

Pranayama（呼吸法）
├── name, instructions, benefits
└── contraindications

Meditation（冥想）
├── type, guidedSteps
└── duration, scenario

Philosophy（哲学）
├── topic, tradition
└── content

Anatomy（解剖）
├── bodyPart, muscleGroup
└── relatedAsanas

Course（课程）
├── teacher, type, level
├── sequence: Asana[]（有序体式列表）
├── duration, schedule
└── location

LessonPlan（教案）
├── teacher, course
├── sequence + teachingNotes
├── cueing: 口令提示
└── visibility: private/public
```

## 技术方案要点

### 内容管理

- 知识库内容使用 **MDX** 编写，存放在项目仓库中
- 体式、呼吸法等用 frontmatter 定义结构化元数据
- 教练信息同样用 MDX + frontmatter 管理
- 通过 Next.js 的静态生成（SSG）构建页面

### 示例：体式 MDX 文件结构

```mdx
---
id: warrior-ii
nameZh: 战士二式
nameEn: Warrior II
nameSanskrit: Virabhadrasana II
category: [站立, 开髋]
level: 入门
muscles: [股四头肌, 臀中肌, 三角肌]
contraindications: [膝关节损伤, 高血压（手臂不举过头顶）]
relatedAsanas: [warrior-i, extended-side-angle]
relatedPranayama: [ujjayi]
---
```

## 设计原则

- **内容优先**：知识库的内容质量是核心竞争力，技术服务于内容的结构化与呈现
- **渐进增强**：第一期纯内容展示，不引入数据库和用户系统；二期按需增加
- **MDX 驱动**：教练和学员都不需要 CMS 后台，内容以 MDX 文件形式管理，版本可控
- **关联发现**：知识不是孤岛，体式、解剖、呼吸法之间的关联关系是核心价值
- **移动友好**：虽然是 Web 站点，但瑜伽场景下手机浏览频率很高，移动端体验需优先保证
